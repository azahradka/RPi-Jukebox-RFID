# PN532 I2C Reader

The PN532-based RFID/NFC reader connected via I2C

This reader module is based on the [py532lib
library](https://github.com/HubCityLabs/py532lib) and uses the I2C bus.
It utilizes a polling mechanism. It only needs 2 pins, but has a certain
performance overhead due to polling.

## Installation

### Prerequisites

1. **Enable I2C interface:**
```bash
sudo raspi-config nonint do_i2c 0
sudo reboot
```

2. **Install py532lib in virtual environment:**
```bash
cd ~/RPi-Jukebox-RFID
source .venv/bin/activate
pip install py532lib
```

3. **Verify I2C device is detected:**
```bash
sudo i2cdetect -y 1
```
Should show device at address `24` (0x24).

## Configuration

### Module Name

**Important:** The module name must be `pn532_i2c_py532` (not `pn532_i2c`).

### Configuration Format

The RFID configuration file (`shared/settings/rfid.yaml`) must use a **dictionary format** with reader keys like `read_00`:

```yaml
rfid:
  readers:
    read_00:
      module: pn532_i2c_py532
      config:
        i2c_bus: 1
        i2c_address: 0x24
        log_all_cards: false
      same_id_delay: 1.0
      log_ignored_cards: false
      place_not_swipe:
        enabled: true
        card_removal_action:
          alias: pause
```

**Common mistake:** Using a list format with `- name:` will cause the reader to fail loading.

## Options

### Reader-Specific Options

- **`log_all_cards`** (boolean, default: false)
  When `true`, logs every card detection with the card ID. Useful for debugging.
  Example log: `Card detected with ID = 2753298439`

- **`i2c_bus`** (integer, default: 1)
  I2C bus number. On most Raspberry Pi models, this is bus 1.

- **`i2c_address`** (hex, default: 0x24)
  I2C address of the PN532. The default address is 0x24.

### Global RFID Options

These options apply to all RFID readers:

- **`same_id_delay`** (float, seconds, default: 1.0)
  Minimum time between repeated reads of the same card. Prevents accidental duplicate triggers.

- **`log_ignored_cards`** (boolean, default: false)
  When `true`, logs card reads that are ignored due to `same_id_delay`.
  Example log: `Ignoring card id 2753298439 due to same-card-delay (1.0s)`

- **`place_not_swipe.enabled`** (boolean, default: false)
  When `true`, enables place-and-remove mode:
  - Placing card triggers action
  - Removing card triggers removal action (e.g., pause)

- **`place_not_swipe.card_removal_action`** (RPC command)
  Action to trigger when card is removed (only if `place_not_swipe.enabled` is true).
  Example: `{alias: pause}` to pause playback when card is removed.

## Hardware

This reader module has been tested with the excellent and highly
recommendable [Adafruit PN532 breakout
board](https://www.adafruit.com/product/364). It also comes with [heaps
of documentation and even
schematics](https://learn.adafruit.com/adafruit-pn532-rfid-nfc/downloads).

I have used various RFID boards and this one has an excellent reading
range and no problem with various types of cards and stickers.

You can usually pick up a board at

* <https://www.berrybase.de/sensoren-module/rfid-nfc/pn532-nfc/rfid-controller-breakout-board>
* <https://shop.pimoroni.com/products/adafruit-pn532-nfc-rfid-controller-shield-for-arduino-extras>

## Board Connections

### Default wiring

| PN532 | RPI GPIO     | RPI Pin |
|-------|--------------|---------|
| 5V    | 5V           | > 4     |
| GND   | GND          | > 6     |
| SDA   | GPIO 2 (SDA) | > 3     |
| SCL   | GPIO 3 (SCL) | > 5     |

Other pins from the Adafruit board need not be connected. Especially do
**not** connect the 3.3V pin, if connecting the 5.0V pin. It is
recommended to use the 5.0V because that does not draw power from the
PI's own voltage regulator.

## Jumpers

### Jumper settings for I2C protocol

| Jumper | Position |
|--------|----------|
|SEL0    | ON       |
|SEL1    | OFF      |

## Troubleshooting

### Reader Not Loading

**Symptom:** Jukebox starts but RFID reader doesn't work. RPC calls to `rfid.reader.*` fail with "Plugin object 'reader' not registered".

**Common causes:**

1. **Missing py532lib library:**
```bash
cd ~/RPi-Jukebox-RFID
source .venv/bin/activate
python -c "from py532lib.mifare import Mifare; print('OK')"
```
If import fails, install: `pip install py532lib`

2. **Wrong module name in config:**
Check `shared/settings/rfid.yaml`:
```yaml
# ❌ WRONG:
module: pn532_i2c

# ✅ CORRECT:
module: pn532_i2c_py532
```

3. **Wrong YAML format (list instead of dictionary):**
```yaml
# ❌ WRONG - List format:
readers:
  - name: PN532_I2C
    module: pn532_i2c_py532

# ✅ CORRECT - Dictionary format:
readers:
  read_00:
    module: pn532_i2c_py532
```

4. **I2C device not detected:**
```bash
sudo i2cdetect -y 1
```
If no device at 0x24:
- Check wiring
- Verify jumpers: SEL0=ON, SEL1=OFF
- Enable I2C: `sudo raspi-config nonint do_i2c 0 && sudo reboot`

### Card Not Detected

**Symptom:** Card placement doesn't trigger any action or log message.

**Solutions:**

1. **Enable verbose logging:**
```yaml
# In shared/settings/rfid.yaml:
config:
  log_all_cards: true
log_ignored_cards: true
```
Restart jukebox, place card, check logs for "Card detected with ID = ..."

2. **Verify reader initialized:**
```bash
journalctl --user -u jukebox-daemon | grep -i 'pn532'
```
Should show: "Initializing reader 'PN532 reader via I2C using py532 library'"

3. **Test hardware directly:**
```bash
cd ~/RPi-Jukebox-RFID
source .venv/bin/activate
python << 'EOF'
from py532lib.mifare import Mifare
device = Mifare()
device.SAMconfigure()
print("Place card on reader...")
card = device.scan_field()
print(f"Card UID: {int(card.hex(), 16)}" if card else "No card")
EOF
```

### Performance Tuning

**Polling frequency:** The PN532 polls for cards continuously. On slower devices (Pi Zero), this can consume ~5-10% CPU.

**Reduce CPU usage:**
- Use `place_not_swipe: false` if you don't need removal detection
- Increase `same_id_delay` if cards are accidentally retriggering

## Logging

When `log_all_cards: true` is enabled, every card detection is logged:

```
Card detected with ID = 2753298439
Received card id = '2753298439'
Unknown card: '2753298439'  (or action taken if registered)
```

When `log_ignored_cards: true` is enabled, repeated reads within `same_id_delay` are logged:

```
Ignoring card id 2753298439 due to same-card-delay (1.0s)
```

This helps debug cards that aren't triggering as expected.
